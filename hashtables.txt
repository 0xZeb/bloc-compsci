> 1. What is a hash table?

A hash table is a data struture that stores key/value pairs. Keys may exist only once per map.

> 2. What is hashing?

Hashing is the process of creating a unique key  with a hash function

> 3. How does a hash table store data?

After encoding an unique key, the pair and key are stored in the map or hash table
at the location corresponding to the hash code.

> 4. How are hash tables and objects different?

They are the same thing. Internal lookups are performed when accessing an objects methods and properties...sounds like a hash table.
